# GRPO (Group Relative Policy Optimization) 算法

本项目实现了 **GRPO** 算法。该算法由 DeepSeek 团队在其模型对齐（Alignment）训练中大规模使用，作为 PPO 的一种更高效、硬件更友好的替代方案。

## 目录结构

| 文件 | 说明 |
| :--- | :--- |
| `grpo_agent.py` | **智能体定义**：实现了基于组（Group）的相对优势估计。 |
| `train_grpo.py` | **训练脚本**：使用简单的数学符号任务演示 GRPO 的对齐能力。 |

## 核心算法原理

### 1. 摆脱 Critic 网络
传统的 PPO 需要一个庞大的 Critic 网络（通常与 Policy 网络等大）来估计状态价值 $V(s)$，这会消耗双倍的显存。

GRPO 的核心创新在于：**通过对同一个状态采样一组动作（outputs），并根据这一组动作的奖励均值和标准差来直接估计优势。**
$$ A_{i} = \frac{R_i - \text{mean}(R_{group})}{\text{std}(R_{group})} $$

### 2. 相对性
这种相对性使得模型能够自动识别出一组回答中哪些是优秀的，哪些是平庸的，从而在不需要显式价值预测的情况下实现策略提升。

### 3. KL 约束
为了防止策略更新过快导致模型崩溃（崩溃），GRPO 依然保留了 KL 散度作为惩罚项，确保新策略不会偏离参考策略太远。

## 使用方法

### 运行训练
```bash
python train_grpo.py
```
虽然演示任务很简单（数学求和），但该算法的逻辑与 DeepSeek-R1 等顶级模型中所使用的逻辑完全一致。
